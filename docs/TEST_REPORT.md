*   **功能：`getUniqueSelector` 選擇器生成**
    *   **測試場景**：傳入頁面上的 H1, P, 包含在 Card 內的 H3。
    *   **預期結果**：生成具備父層上下文且唯一的 CSS 選擇器 (如 `.card:nth-child(1) > h3`)。
    *   **結果**：✅ 通過。生成之選擇器能準確定位到指定元素。
*   **功能：`autoSave` 草稿儲存**
    *   **測試場景**：修改 H1 內容後等待 5 秒。
    *   **預期結果**：`localStorage` 中的 `page_draft_index.html` 鍵值應包含修改後的 HTML 字串。
    *   **結果**：✅ 通過。草稿準確儲存。

## 2. 整合測試 (Integration Test) - 流程驗證
*   **流程：編輯 → 自動存檔 → 發布 → 正式渲染**
    *   **測試場景**：在首頁修改 Vincent 為 "Admin Vincent"，等待 5 秒後點擊發布。
    *   **驗證步驟**：
        1. 檢查 `localStorage` 草稿內容。
        2. 輸入正確 PIN (8888) 進行發布。
        3. 重新載入頁面，檢查 H1 內容。
    *   **結果**：✅ 通過。正式環境內容已更新為 "Admin Vincent"。

## 3. 權限與安全性測試 (Permission & Security)

    *   **結果**：✅ 通過。`initLiveEdit` 中的角色檢查機制運作正常。

    *   **預期結果**：系統應提示驗證失敗，且不更新 `localStorage` 正式環境內容。
    *   **結果**：✅ 通過。內容未被修改。
